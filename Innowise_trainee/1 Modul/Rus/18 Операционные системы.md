## Что такое операционная система

**Операционная система (OS)** — это основной слой ПО между hardware и приложениями. Она управляет ресурсами компьютера и даёт программам способ работать с файлами, памятью, устройствами и пользовательским интерфейсом.

**Цель:** позволить приложениям работать безопасно и предсказуемо.

**Основные функции OS:**
- управление процессами
- управление памятью
- управление файлами
- управление устройствами
- учётные записи и permissions
- работа с сетью

**Примеры:** Windows, Linux, macOS, Android, iOS.

> [!danger] Главное
> Приложения обычно не работают с hardware напрямую. Они обращаются к операционной системе.

## Архитектура операционной системы

### Kernel

**Kernel** — ядро операционной системы. Оно работает ближе всего к hardware и управляет низкоуровневыми операциями.

**Цель:** управлять CPU, памятью, устройствами и system calls.

**Пример:** когда приложение хочет прочитать файл, оно обращается к kernel через API операционной системы.

### User space

**User space** — область, где работают обычные приложения. Браузеры, редакторы, test tools и terminal обычно работают здесь.

**Цель:** изолировать приложения от критичных частей системы, чтобы сбой одного приложения не ломал всю OS напрямую.

**Пример:** Chrome падает в user space, но вся операционная система обычно продолжает работать.

### File system

**File system** — структура OS для хранения и организации файлов и папок.

**Цель:** хранить данные в предсказуемой иерархии и контролировать доступ к ним.

**Распространённые примеры:**
- **Windows:** NTFS
- **Linux:** ext4
- **macOS:** APFS

**Применимость для QA:**
- пути к файлам отличаются между OS
- permissions могут блокировать тесты
- чувствительность к регистру может менять поведение приложения

> [!tip] Разница путей
> Windows обычно использует обратные слэши, например `C:\Users\John\file.txt`. Linux и macOS используют прямые слэши, например `/Users/john/file.txt`.

## Windows vs Linux vs macOS

Эти системы решают одну и ту же базовую задачу, но отличаются дефолтным поведением, инструментами и особенностями окружения.

| Область | Windows | Linux | macOS |
|---|---|---|---|
| Основная аудитория | Обычные пользователи, enterprise | Серверы, разработчики, power users | Пользователи экосистемы Apple |
| Разделитель путей | `\` | `/` | `/` |
| Чувствительность к регистру | Обычно нет | Да | Обычно нет по умолчанию |
| Установка пакетов | MSI, EXE, Store, winget | apt, yum, dnf, pacman | App Store, DMG, Homebrew |
| GUI-стиль | Start menu, Taskbar | Зависит от дистрибутива | Dock, menu bar |
| Terminal | Command Prompt, PowerShell | Shell вроде Bash | Terminal с zsh/bash |

**Применимость для QA:**
- одно и то же приложение может вести себя по-разному на разных OS
- install flow зависит от OS
- file upload, paths и permissions могут ломаться только на одной платформе
- hotkeys и UI-конвенции отличаются

## Основы Windows

Windows — самая распространённая desktop OS во многих QA-проектах, особенно в enterprise-среде.

**Полезные встроенные инструменты:**
- **File Explorer** — просмотр файлов и папок
- **Control Panel / Settings** — управление настройками системы
- **Task Manager** — просмотр процессов, CPU и памяти
- **Event Viewer** — просмотр системных и application logs

### Windows Registry

**Windows Registry** — центральная база данных настроек системы и приложений.

**Цель:** хранить configuration, которую использует Windows и многие установленные программы.

**Как это работает:** настройки организованы в keys и values, похожие на папки и элементы.

**Почему это важно для QA:**
- некоторые приложения хранят configuration в registry
- broken installers могут оставлять некорректные keys
- настройки могут оставаться после uninstall и влиять на повторное тестирование

**Пример:** приложение хранит путь установки или поведение при запуске в registry keys.

### Установка и удаление программ в Windows

Программы в Windows часто устанавливаются через **MSI**, **EXE** или **Microsoft Store**.

**Что проверяет QA:**
- installer запускается корректно
- путь установки можно выбрать, если это ожидается
- создаются shortcuts на desktop или в start menu, если нужно
- uninstall удаляет приложение корректно
- старые config или data не остаются неожиданно

**Частые пути удаления:**
- **Settings -> Apps**
- **Control Panel -> Programs and Features**

## Основы macOS

macOS — desktop operating system от Apple. QA-инженеры встречают её при тестировании браузеров, desktop apps или mobile-related tooling.

**Полезные встроенные инструменты:**
- **Finder** — просмотр файлов и папок
- **System Settings** — управление настройками OS
- **Activity Monitor** — просмотр CPU, памяти, диска и процессов

### Интерфейс macOS vs Windows

Интерфейс macOS отличается от Windows по нескольким частым точкам.

| Элемент | Windows | macOS |
|---|---|---|
| Главный launcher | Start menu | Launchpad / Spotlight |
| Главная панель приложений | Taskbar | Dock |
| Меню приложения | Внутри окна | Глобальная menu bar сверху |
| Кнопки окна | Обычно справа сверху | Слева сверху |

**Почему это важно для QA:**
- пользовательские шаги отличаются между платформами
- screenshots и инструкции должны соответствовать OS
- hotkeys и ожидания от UI разные

### Терминал macOS

**Terminal** — это интерфейс командной строки в macOS.

**Цель:** запускать команды, проверять файлы и автоматизировать работу быстрее, чем через GUI.

**Базовые команды:**
- `pwd` — показать текущую папку
- `ls` — список файлов
- `cd` — перейти в папку
- `cat` — вывести содержимое файла
- `grep` — искать текст

**Применимость для QA:**
- полезен для чтения логов
- полезен для проверки путей и permissions
- часто используется вместе с developer tools и automation

## Меры безопасности операционных систем

Операционные системы содержат встроенные security controls, которые защищают данные и снижают риск malware или несанкционированного доступа.

**Примеры для Windows:**
- **UAC (User Account Control)** — просит подтверждение перед действиями с повышенными правами
- **Windows Defender** — встроенная антивирусная защита
- **Firewall** — управляет сетевым доступом
- **BitLocker** — шифрование диска

**Примеры для macOS:**
- **Gatekeeper** — блокирует недоверенные приложения
- **XProtect** — встроенная защита от malware
- **FileVault** — шифрование диска
- **SIP (System Integrity Protection)** — защищает важные системные файлы

**Примеры для Linux:**
- права доступа к файлам (`rwx`)
- `sudo` для привилегированных действий
- подписанные пакеты из репозиториев

**Применимость для QA:**
- security prompts могут влиять на install и launch flows
- firewall rules могут блокировать соединения приложения
- permissions могут ломать доступ к файлам или настройку тестов

> [!tip] Security vs bug
> Иногда приложение не сломано. Действие специально блокируется security-слоем OS.

## Вопросы для собеседования

### Вопросы начального уровня

**1. Что такое операционная система?**
Операционная система — это основной слой ПО между hardware и приложениями. Она управляет ресурсами, такими как CPU, память, файлы и устройства, и позволяет программам безопасно с ними работать.

**2. Каковы основные функции операционной системы?**
Управление процессами, памятью, файлами, устройствами, сетью и правами пользователей.

**3. В чём разница между kernel и user space?**
Kernel — центральная часть OS, работающая близко к hardware. User space — область, где работают обычные приложения. Такое разделение улучшает стабильность и безопасность.

**4. Что такое file system?**
File system — это структура OS для хранения и организации файлов и папок, например NTFS, ext4 или APFS.

### Вопросы среднего уровня

**1. Какие основные различия между Windows, Linux и macOS важны для QA?**
Они отличаются путями, чувствительностью к регистру, способами установки программ, terminal tools, permissions и поведением UI. Из-за этого баги могут проявляться только на одной OS.

**2. Что такое Windows Registry?**
Windows Registry — это центральная база данных системных и application settings. QA может проверять её при тестировании installers, configuration и cleanup после uninstall.

**3. Почему чувствительность к регистру важна?**
В Linux `Report.txt` и `report.txt` — это разные файлы. В Windows они обычно считаются одним и тем же. Это может вызывать баги только на одной платформе.

**4. Почему QA полезно знать базовые terminal commands?**
Они помогают быстрее проверять logs, paths, files, permissions и состояние окружения, чем только через GUI.

### Вопросы повышенного уровня

**1. Приложение работает на Windows, но падает на Linux при открытии config file. В чём может быть причина?**
Возможные причины: другой разделитель путей, проблемы с правами доступа к файлу или чувствительность к регистру в имени файла.

**2. Почему тестирование uninstall — это не только “приложение исчезло”?**
Потому что могут оставаться registry keys, config files, services или cached data, которые повлияют на следующую установку или дальнейшее тестирование.

### Вопросы с кодом

**1. Что говорит вам такой путь?**

```bash
/Users/john/Documents/report.txt
```

Это Unix-style path, поэтому он характерен для Linux или macOS, а не для обычного Windows-формата пути.
