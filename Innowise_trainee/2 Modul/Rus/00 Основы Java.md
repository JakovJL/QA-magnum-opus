## Что такое Java

Java — высокоуровневый объектно-ориентированный язык. Главная особенность — платформенная независимость: код работает на любом устройстве с JVM.

### Как это работает

1. Вы пишете `.java` файлы
2. **Компилятор** (`javac`) создаёт `.class` файлы с байткодом
3. **JVM** преобразует байткод в машинный код

> [!info] WORA (Write Once, Run Anywhere)
> Байткод одинаков на всех платформах. Нужна только своя JVM под Windows, Linux или macOS.

## JDK vs JRE vs JVM

| Компонент | Что включает |
|---|---|
| **JVM** | Исполняет байткод. Ядро Java |
| **JRE** | JVM + стандартные библиотеки |
| **JDK** | JRE + инструменты разработки (`javac`) |

> [!tip] Простое правило
> Чтобы запускать → JRE. Чтобы писать и компилировать → JDK. JDK уже включает JRE.

## Первая программа на Java

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

**Компиляция и запуск:**
```bash
javac HelloWorld.java   # → HelloWorld.class
java HelloWorld         # → Hello, World!
```

## Пакеты и импорт

**Пакет** — папка для группировки классов. Предотвращает конфликты имён.

```java
package com.innowise.training;
import java.util.Scanner;
```

**Соглашение:** обратное доменное имя. Для Innowise: `com.innowise.training.module2`.

## Соглашения об именах

| Что | Стиль | Пример |
|---|---|---|
| Классы | PascalCase | `HelloWorld`, `LoginPage` |
| Методы | camelCase | `getUserName()` |
| Переменные | camelCase | `totalCount` |
| Константы | UPPER_SNAKE | `MAX_SIZE` |
| Пакеты | lowercase | `com.innowise` |

```java
public class UserAccount {
    private static final int MAX_LOGIN_ATTEMPTS = 3;
    private String userName;

    public String getUserName() { return userName; }
}
```

## Вопросы для собеседования

### Вопросы начального уровня

**1. В чём разница между JDK, JRE и JVM?**
JVM (Java Virtual Machine) исполняет байткод — это ядро среды выполнения. JRE — это JVM + стандартные библиотеки классов (нужна, чтобы *запускать* Java-программы). JDK — это JRE + инструменты разработки вроде `javac` и `jar` (нужен, чтобы *разрабатывать* на Java). JDK включает JRE, поэтому для разработки и запуска достаточно установить JDK.

**2. Что означает «Write Once, Run Anywhere»?**
Исходный код Java компилируется в байткод (`.class` файлы). Байткод не зависит от платформы — он работает на любом устройстве с JVM. Только JVM должна быть платформно-специфичной. Поэтому код пишется один раз и работает на Windows, Linux, macOS и любой другой платформе с JVM.

**3. Что такое пакет в Java?**
Пакет — это пространство имён, которое организует классы. Он предотвращает конфликты имён, управляет доступом (модификатор доступа по умолчанию) и упрощает управление кодом. Пакеты используют обратное доменное имя: `com.innowise.training`.

**4. Какие соглашения об именах в Java?**
Классы — PascalCase (`HelloWorld`). Методы и переменные — camelCase (`getUserName`, `totalCount`). Константы — UPPER_SNAKE_CASE (`MAX_SIZE`). Пакеты — строчные (`com.innowise`).

**5. Что произойдёт, если не указать пакет?**
Класс попадёт в *default package*. Это допустимо для небольших примеров, но в реальных проектах крайне не рекомендуется — классы из разных проектов могут конфликтовать.

**6. Что такое параметр `String[] args` в main?**
Это массив строк, в котором хранятся аргументы командной строки, передаваемые при запуске программы. Например, при `java MyApp arg1 arg2` получится `args[0] = "arg1"` и `args[1] = "arg2"`.

### Вопросы среднего уровня

**1. Почему метод main объявлен как `public static void`?**
`public` — JVM должна иметь к нему доступ извне класса. `static` — JVM вызывает его без создания объекта (объекта ещё нет при запуске программы). `void` — метод main не возвращает значение JVM.

**2. В чём разница между байткодом и машинным кодом?**
Байткод (`.class` файлы) — это промежуточное представление, которое JVM интерпретирует или JIT-компилирует. Он не привязан к конкретному физическому процессору. Машинный код — это нативные бинарные инструкции для конкретной архитектуры CPU (x86, ARM). Цепочка: байткод → JIT-компилятор → машинный код во время выполнения.

**3. В чём разница между `javac` и `java`?**
`javac` — это компилятор Java: он переводит `.java` исходники в `.class` файлы байткода. `java` — это запускатор: он стартует JVM и загружает main-метод указанного класса.

**4. Может ли в одном `.java` файле быть несколько public классов?**
Нет. В одном исходном файле может быть максимум один `public` класс, и имя файла должно совпадать с именем этого класса. Несколько non-public классов в одном файле допустимы.

### Вопросы повышенного уровня

**1. Можно ли сделать метод main не public?**
Да, он скомпилируется. Но JVM не найдёт его при запуске и выбросит `NoSuchMethodError` во время выполнения. JVM ищет именно `public static void main(String[])`.

**2. Можно ли скомпилировать и запустить Java-код без main?**
Скомпилировать — да. Класс без main компилируется нормально. Запустить — нет, JVM нужна точка входа. Однако класс можно запустить без main через JUnit или если на него ссылается другой класс, у которого main есть.

**3. Если класс успешно скомпилирован, но `.class` файл удалён — может ли класс работать?**
Нет. JVM исполняет `.class` файлы. Если байткод отсутствует, JVM не может загрузить класс. Однако, если класс уже был загружен в память ClassLoader'ом до удаления файла, он продолжит работать — JVM читает `.class` файл только один раз, во время загрузки класса.

**4. Почему Java использует и компилятор, и интерпретатор?**
Java использует компилятор (`javac`), чтобы получить байткод (переносимый, платформенно-независимый). Во время выполнения JVM сначала интерпретирует байткод, а для горячих участков (hot paths) включает JIT-компилятор, переводящий их в нативный машинный код ради производительности. Такой гибрид даёт и переносимость, и скорость.

### Вопросы с кодом

**1. Объясните каждую строку этой программы.**

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

- `public class HelloWorld` — объявляет класс; файл должен называться `HelloWorld.java`
- `public static void main(String[] args)` — точка входа; JVM вызывает его без объекта (`static`), извне класса (`public`), и он ничего не возвращает (`void`)
- `String[] args` — аргументы командной строки
- `System.out.println(...)` — печатает в консоль

**2. Скомпилируется ли этот код? Если да, что произойдёт при выполнении?**

```java
// Файл: App.java
public class App {
    private static void main(String[] args) {
        System.out.println("Hi");
    }
}
```

Скомпилируется — видимость `main` не проверяется на этапе компиляции. При выполнении JVM не находит `public static void main(String[])` и выбрасывает `NoSuchMethodError`. Исправление — сделать `main` `public`.
