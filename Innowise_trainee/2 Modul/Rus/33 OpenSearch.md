## Что такое OpenSearch

**OpenSearch** — поисковая система и хранилище данных, оптимизированное для быстрого поиска, фильтрации и аналитики.

**Цель:** быстро находить документы по тексту и полям, когда обычная база данных не подходит для сложного поиска.

OpenSearch часто используют для поиска товаров, логов, событий и аналитических данных.

---

## Индекс и документ

- **Index** — набор документов одного типа; похож на таблицу по назначению, но устроен иначе;
- **Document** — единица данных в формате JSON;
- **Mapping** — описание полей и их типов;
- **Shard** — часть индекса, которая помогает распределять данные;
- **Replica** — копия shard для отказоустойчивости и чтения.

```json
{
  "id": 42,
  "name": "Wireless mouse",
  "category": "accessories",
  "price": 25.50
}
```

## Полнотекстовый поиск

При полнотекстовом поиске строка разбирается на части — токены. Поэтому поиск по полю `text` может находить разные формы слова, в зависимости от настроенного анализатора.

Для точной фильтрации обычно используют поле типа `keyword`, а для поиска по тексту — поле типа `text`.

> [!tip] `text` и `keyword`
>
> | Тип поля | Для чего подходит |
> |---|---|
> | `text` | Поиск по словам в названии или описании |
> | `keyword` | Точное значение, фильтр, сортировка, агрегация |

## Индексация данных

Данные могут попадать в OpenSearch напрямую через API или асинхронно из основной базы данных через очередь и отдельный сервис индексации.

Во втором случае возникает задержка: запись уже есть в основной системе, но ещё не появилась в поиске. Это называется eventual consistency — согласованность достигается не мгновенно.

Тест должен учитывать это требование: ожидать появление документа ограниченное время, а не делать одну мгновенную проверку.

---

## Что проверять тестировщику

- появился ли в индексе новый или обновлённый документ;
- правильно ли перенесены поля и их типы;
- находится ли документ по нужному запросу;
- не находится ли по запросу, который не должен подходить;
- корректны ли фильтры, сортировка и пагинация;
- исчезает ли документ из поиска после удаления или скрытия;
- соответствует ли задержка индексации требованиям.

## Типичные ошибки

| Ошибка | Как сделать лучше |
|---|---|
| Проверять поиск сразу после записи | Ждать появления документа в пределах согласованного тайм-аута |
| Ожидать точный поиск от поля `text` | Для точного сравнения использовать `keyword` |
| Проверять только число результатов | Проверять также содержимое и порядок результатов |
| Считать индекс основной базой данных | Сверять источник данных и поисковый индекс отдельно |
