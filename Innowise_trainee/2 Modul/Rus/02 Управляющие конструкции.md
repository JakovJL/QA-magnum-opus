## if-else

```java
int age = 18;
if (age >= 18) {
    System.out.println("Можно голосовать");
}

int temperature = 30;
if (temperature > 25) {
    System.out.println("Жарко");
} else {
    System.out.println("Не жарко");
}

int score = 85;
if (score >= 90) {
    System.out.println("A");
} else if (score >= 75) {
    System.out.println("B");
} else {
    System.out.println("F");
}
```

> [!tip] Фигурные скобки
> Всегда используй `{}` даже для одной строки.

## switch

```java
int day = 3;
String dayName;
switch (day) {
    case 1: dayName = "Пн"; break;
    case 2: dayName = "Вт"; break;
    case 3: dayName = "Ср"; break;
    case 4: dayName = "Чт"; break;
    case 5: dayName = "Пт"; break;
    case 6:
    case 7: dayName = "Вых"; break;
    default: dayName = "?";
}
```

### Улучшенный switch (Java 14+)

```java
String dayType = switch (day) {
    case 1, 2, 3, 4, 5 -> "Будний";
    case 6, 7 -> "Выходной";
    default -> "?";
};
```

## Цикл for

```java
for (int i = 0; i < 5; i++) {
    System.out.println("Счёт: " + i);
}

// Обратный отсчёт
for (int i = 5; i > 0; i--) { }

// Шаг 2
for (int i = 0; i <= 10; i += 2) { }

// For-each (когда индекс не нужен)
int[] numbers = {10, 20, 30, 40};
for (int num : numbers) {
    System.out.println(num);
}
```

## while и do-while

```java
// while — проверка ДО, может не выполниться
int i = 0;
while (i < 5) { System.out.println(i); i++; }

// do-while — проверка ПОСЛЕ, выполняется хотя бы раз
int j = 0;
do { System.out.println(j); j++; } while (j < 5);
```

## break и continue

```java
// break — выход из цикла
for (int num : numbers) {
    if (num < 0) break;     // остановиться на первом отрицательном
}

// continue — пропустить итерацию
for (int i = 0; i <= 10; i++) {
    if (i % 2 != 0) continue;  // пропустить нечётные
    System.out.println(i);       // 0 2 4 6 8 10
}
```

## Проход по массивам

```java
int[] scores = {85, 90, 78, 92, 88};

for (int i = 0; i < scores.length; i++) {
    System.out.println("Оценка " + i + ": " + scores[i]);

int sum = 0;
for (int score : scores) { sum += score; }
double average = (double) sum / scores.length;
```

## Вопросы для собеседования

### Вопросы начального уровня

**1. В чём разница между циклами `for` и `while`?**
Используй `for`, когда количество итераций известно заранее (`for (int i = 0; i < 10; i++)`). Используй `while`, когда количество итераций зависит от условия, которое может меняться во время выполнения (`while (input != null)`).

**2. В чём разница между `break` и `continue`?**
`break` немедленно выходит из цикла. `continue` пропускает остаток текущей итерации и переходит к следующей.

**3. Чем `switch` отличается от `if-else`?**
`switch` чище и удобнее, когда одну переменную сравнивают со множеством конкретных значений. `if-else` лучше для проверки диапазонов, булевых условий или сложных логических выражений.

**4. Что такое тернарный оператор и когда его использовать?**
Тернарный оператор (`условие ? еслиИстинно : еслиЛожно`) — это компактная замена `if-else`, возвращающая значение. Используй его для простых бинарных условий. Избегай вложенных тернарных операторов — это вредит читаемости.

**5. В чём разница между `while` и `do-while`?**
`while` проверяет условие *до* тела — он может не выполниться ни разу. `do-while` проверяет условие *после* тела — он выполняется хотя бы один раз. Используй `do-while`, когда тело должно отработать минимум один раз (например, валидация ввода).

### Вопросы среднего уровня

**1. Что такое fall-through в `switch`?**
Fall-through возникает, когда блок `case` не заканчивается `break`. Выполнение переходит в следующий `case`, независимо от того, совпадает ли его значение. Это может быть сделано намеренно (группировка case), но обычно это баг.

**2. Что такое улучшенный `switch` (Java 14+)?**
Улучшенный switch использует синтаксис со стрелкой (`->`) и не требует `break`. Его можно использовать как выражение, возвращающее значение: `String result = switch (x) { case 1 -> "one"; default -> "other"; };`.

**3. Как работает цикл for-each? Можно ли изменять элементы внутри него?**
For-each проходит по массиву или Iterable без использования индекса. Переменная цикла — это копия элемента, поэтому её переназначение (`num = 99`) не меняет массив. Однако, если элементы — объекты, ты *можешь* менять их состояние через переменную цикла (`user.setName("new")`). В коллекциях во время итерации *нельзя* добавлять или удалять элементы (будет выброшено `ConcurrentModificationException`).

**4. Что такое бесконечный цикл? Как его создать намеренно?**
Бесконечный цикл работает вечно, пока его не прервут. `while (true) { }` или `for (;;) { }`. Используется в серверных циклах, слушателях событий или ожидании внешних событий с break-условием.

### Вопросы повышенного уровня

**1. Что такое labeled break в Java?**
Labeled break выходит из указанного внешнего цикла: `outer: for (...) { for (...) { break outer; } }`. Без метки `break` выходит только из самого внутреннего цикла.

**2. Можно ли объявить переменную внутри `for` и использовать её снаружи?**
Нет. Переменная цикла ограничена блоком цикла. `for (int i = 0; ...) { }` — `i` недоступен после цикла. Объяви переменную до цикла, если она нужна позже.

**3. В чём разница между `++i` и `i++` в шаге цикла for?**
При стандартном использовании (`for (int i = 0; i < 5; i++)`) никакой практической разницы нет, потому что обновление происходит после каждой итерации и его значение не используется. Разница появляется, когда инкремент является частью выражения, например `while (i++ < 5)` vs `while (++i < 5)`.

### Вопросы с кодом

**1. Что выведет этот код и почему?**

```java
for (int i = 0; i < 5; i++) {
    if (i == 3) break;
    System.out.print(i);
}
```

Вывод: `012`. Цикл останавливается при `i == 3`, потому что `break` немедленно выходит из цикла — выводятся только 0, 1 и 2.

**2. Что выведет этот код и почему?**

```java
for (int i = 0; i < 5; i++) {
    if (i == 3) continue;
    System.out.print(i);
}
```

Вывод: `0124`. `continue` пропускает остаток итерации при `i == 3`, поэтому 3 не выводится, но цикл продолжается с 4.

**3. Что выведет этот switch при `day == 1`?**

```java
switch (day) {
    case 1: System.out.print("one ");
    case 2: System.out.print("two "); break;
    default: System.out.print("def");
}
```

Выведет `one two ` — fall-through. В `case 1` нет `break`, поэтому выполнение переходит в `case 2`. А вот `break` после `case 2` уже останавливает выполнение.
