## Основы наследования

Потомок переиспользует поля и методы родителя. Ключевое слово — `extends`.

```java
public class Animal {
    protected String name;
    public void eat() { System.out.println(name + " ест"); }
}

public class Dog extends Animal {
    public void bark() { System.out.println(name + " гавкает"); }
}

Dog dog = new Dog();
dog.name = "Rex";
dog.eat();      // унаследовано
dog.bark();     // своё
```

> [!info] Одиночное наследование
> Класс может расширять только одного родителя. Многоуровневость допустима.

## super

```java
public class Vehicle {
    public Vehicle(String brand) { }
}

public class Car extends Vehicle {
    public Car(String brand) {
        super(brand);       // должна быть первой строкой
    }
}
```

## Переопределение (overriding)

Потомок заново определяет метод родителя. Та же сигнатура.

```java
public class Shape {
    public double getArea() { return 0; }
}

public class Circle extends Shape {
    @Override
    public double getArea() { return Math.PI * radius * radius; }
}
```

---

## Перегрузка (overloading)

Одно имя, разные параметры в одном классе.

```java
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}
```

| Использование | Эффект |
|---|---|
| `final` переменная | Константа |
| `final` метод | Нельзя переопределить |
| `final` класс | Нельзя наследовать |

## Абстрактные классы

Нельзя создать напрямую. Могут содержать абстрактные методы (без тела).

```java
public abstract class Payment {
    public abstract void processPayment();
    public void printReceipt() { System.out.println("Платёж обработан"); }
}

public class CreditCardPayment extends Payment {
    @Override
    public void processPayment() {
        System.out.println("Оплата картой");
    }
}
```

## Полиморфизм

Один вызов — разное поведение в зависимости от реального типа объекта.

```java
Payment p1 = new CreditCardPayment(100, "1234");
Payment p2 = new PayPalPayment(50, "email");
p1.processPayment();   // версия CreditCardPayment
p2.processPayment();   // версия PayPalPayment
```

## instanceof

Проверяет, является ли объект экземпляром класса.

```java
if (payment instanceof CreditCardPayment) {
    CreditCardPayment cc = (CreditCardPayment) payment;
}
```

> [!caution] Запах кода
> Частое instanceof значит, что дизайн можно улучшить. Предпочитай полиморфизм.

## Вопросы для собеседования

### Вопросы начального уровня

**Что такое наследование в Java? Какое ключевое слово используется?**
Наследование позволяет классу-потомку переиспользовать поля и методы родительского класса. Ключевое слово — `extends`. Java поддерживает одиночное наследование — класс может расширять только один родительский класс.

**Что делает ключевое слово `super`?**
`super` ссылается на родительский класс. Используется, чтобы: (1) вызвать конструктор родителя (`super(params)` — должен быть первой строкой в конструкторе потомка), (2) обратиться к методам родителя при переопределении (`super.methodName()`).

**В чём разница между абстрактным и конкретным классом?**
Абстрактный класс (объявлен с `abstract`) нельзя инстанциировать напрямую. Он может содержать абстрактные методы (без тела), которые потомки обязаны реализовать. Конкретный класс можно инстанциировать, и он обязан реализовать все унаследованные абстрактные методы.

**Что такое аннотация `@Override`? Зачем её использовать?**
`@Override` сообщает компилятору, что метод предназначен для переопределения родительского метода. Она необязательна, но крайне рекомендуется — компилятор ловит ошибки вроде опечаток в имени метода или неверных типов параметров.

**Что такое полиморфизм? Приведите пример.**
Полиморфизм означает «много форм» — один и тот же вызов метода ведёт себя по-разному в зависимости от фактического типа объекта во время выполнения. Пример: `Animal a = new Dog(); a.makeSound();` вызовет версию `makeSound()` из `Dog`, а не из `Animal`.

**Что делает ключевое слово `final` при применении к методу? К классу? К переменной?**
`final` метод — нельзя переопределить в потомках. `final` класс — нельзя наследовать (расширять). `final` переменная — нельзя переназначить (константа).

**Что такое оператор `instanceof`?**
`instanceof` проверяет, является ли объект экземпляром конкретного класса, подкласса или интерфейса. Возвращает `boolean`. Используется перед кастом, чтобы избежать `ClassCastException`.

### Вопросы среднего уровня

**В чём разница между перегрузкой методов (overloading) и переопределением (overriding)?**
Перегрузка — одно имя, разные параметры, в том же классе (compile-time полиморфизм). Переопределение — та же сигнатура, в классе-потомке, заменяет родительский метод (runtime полиморфизм).

**В чём разница между `extends` и `implements`?**
`extends` — для наследования класс-класс (или интерфейс-интерфейс). Класс может расширять только один класс. `implements` — для выполнения классом контракта интерфейса. Класс может реализовать несколько интерфейсов.

**Почему Java не поддерживает множественное наследование классов?**
Чтобы избежать «проблемы ромба» (diamond problem): если у двух родительских классов есть одинаковый метод, потомок не знает, какую версию наследовать. Java решает это через интерфейсы — класс может реализовать несколько интерфейсов.

**Почему чрезмерное использование `instanceof` считается плохим дизайном?**
Потому что это обходит полиморфизм. Вместо проверки типа и ручного каста (`if (animal instanceof Dog) { ((Dog) animal).bark(); }`) следует позволить объекту делать работу самому (`animal.makeSound()`). Частый `instanceof` означает, что иерархию, скорее всего, нужно перепроектировать.

**Может ли конструктор быть `final`? Может ли он быть `static`?**
Нет в обоих случаях. Конструкторы — не обычные методы, их нельзя переопределить или унаследовать, поэтому `final` не применим. Конструкторы не могут быть `static`, потому что они должны быть связаны с созданием объекта.

**Может ли абстрактный класс иметь конструктор? Каково его назначение?**
Да. Абстрактный класс может иметь конструктор, и он вызывается через `super()` из конструктора потомка. Его назначение — инициализировать общее состояние (поля) абстрактного класса до того, как выполнится конструктор потомка.

### Вопросы повышенного уровня

**Что будет, если вызвать `super.super.methodName()`?**
Ошибка компиляции. Java не разрешает обращаться к методам «дедушки» напрямую, если родитель их переопределил. Через `super` доступен только прямой родитель. Чтобы получить версию дедушки, родитель должен сам её открыть (экспонировать).

**Можно ли переопределить `private` метод?**
Нет. Private-методы не наследуются, поэтому их нельзя переопределить. Если объявить метод с тем же именем в потомке, это *новый* метод, а не переопределение — и если в потомке добавить `@Override`, компилятор сообщит об ошибке.

**Каков порядок инициализации при загрузке и родителя, и потомка?**
Сначала static родителя (поля и блоки), затем static потомка. На `new Child()`: instance-поля и блоки родителя, затем конструктор родителя, затем instance-поля и блоки потомка, затем конструктор потомка. `super()` должен быть первым оператором в конструкторе потомка.

### Вопросы с кодом

```java
class Parent {
    public void display() { System.out.println("Parent display"); }
}

class Child extends Parent {
    @Override
    public void display() {
        super.display();
        System.out.println("Child display");
    }
}

public class Main {
    public static void main(String[] args) {
        Parent obj = new Child();
        obj.display();
    }
}
```
**Что выведет этот код и почему?**
**Ответ:**
```
Parent display
Child display
```
Тип ссылки — `Parent`, но объект — `Child`, поэтому runtime-полиморфизм вызывает `display()` из `Child`. Внутри него `super.display()` сначала выполняет родительскую версию, затем выводится "Child display".

```java
class Payment {
    public void processPayment() { System.out.println("Processing payment"); }
}
class CreditCardPayment extends Payment {
    @Override
    public void processPayment() { System.out.println("Processing via card"); }
}

Payment p = new CreditCardPayment();
((CreditCardPayment) p).processPayment();
```
**Скомпилируется и запустится ли это? Что выведет?**
**Ответ:** Компилируется и запускается, выводит `Processing via card`. В `p` на самом деле лежит `CreditCardPayment`, поэтому нисходящий каст безопасен и вызывается переопределённый метод.

```java
public class Vehicle {
    protected String brand;
    public Vehicle(String brand) { this.brand = brand; }
}

public class Car extends Vehicle {
    private String model;
    public Car(String brand, String model) {
        this.model = model;
    }
}
```
**Скомпилируется ли это? Если нет, почему?**
**Ответ:** Нет. У `Vehicle` нет конструктора без параметров, поэтому конструктор потомка обязан явно вызвать `super(brand)` первой строкой. Без этого компилятор пытается вставить дефолтный вызов `super()`, который не срабатывает, потому что подходящего родительского конструктора нет. Исправление: добавить `super(brand);` перед `this.model = model;`.
