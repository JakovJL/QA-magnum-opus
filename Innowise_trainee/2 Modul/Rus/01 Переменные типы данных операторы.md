## Примитивные типы данных

В Java 8 примитивных типов. Они хранят значения напрямую в памяти.

| Тип | Размер | Диапазон | По умолч. | Пример |
|---|---|---|---|---|
| `byte` | 1 байт | -128 до 127 | 0 | `byte b = 100;` |
| `short` | 2 байта | -32 768 до 32 767 | 0 | `short s = 1000;` |
| `int` | 4 байта | ~-2.1 млрд до 2.1 млрд | 0 | `int age = 25;` |
| `long` | 8 байт | очень большой | 0L | `long pop = 8_000_000_000L;` |
| `float` | 4 байта | ~±3.4E+38 | 0.0f | `float pi = 3.14f;` |
| `double` | 8 байт | ~±1.7E+308 | 0.0 | `double price = 19.99;` |
| `boolean` | 1 бит | true / false | false | `boolean isReady = true;` |
| `char` | 2 байта | один символ Unicode | '\u0000' | `char grade = 'A';` |

> [!tip] Нижние подчёркивания в числах
> Можно использовать `_` для читаемости: `int million = 1_000_000;` Java игнорирует подчёркивания.

```java
public class PrimitiveDemo {
    public static void main(String[] args) {
        int age = 25;
        double price = 19.99;
        boolean isStudent = true;
        long worldPopulation = 8_000_000_000L;
    }
}
```

## Ссылочные типы

Хранят **адрес в памяти** на объект, а не само значение.

| Тип | Описание | Пример |
|---|---|---|
| `String` | Текст (неизменяемый) | `String name = "Alice";` |
| Массивы | Коллекция одного типа | `int[] numbers = {1, 2, 3};` |
| Классы | Пользовательские типы | `Scanner input = new Scanner(System.in);` |

### Примитивы vs Ссылки

```java
// Примитив — копирует ЗНАЧЕНИЕ
int a = 5;
int b = a;
b = 10;         // a всё ещё 5

// Ссылка — копирует АДРЕС (оба указывают на один объект)
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;
arr2[0] = 99;   // arr1[0] теперь тоже 99!
```

> [!warning] Ловушка ссылок
> Когда присваиваешь одну ссылочную переменную другой, они указывают на один объект. Изменение через одну затрагивает другую.

## Объявление и инициализация переменных

```java
int count = 10;                     // 1. Объявление + инициализация
int count; count = 10;              // 2. Раздельно
int x = 1, y = 2, z = 3;           // 3. Несколько одной строкой

int value;
// System.out.println(value);       // ❌ Ошибка: не инициализирована
value = 42;
System.out.println(value);          // ✅ OK
```

## Приведение типов (casting)

### Неявное (расширение) — безопасно, автоматически

```java
int myInt = 100;
long myLong = myInt;      // int → long, OK
```

**Порядок:** `byte → short → int → long → float → double`

### Явное (сужение) — может потерять данные

```java
double price = 19.99;
int rounded = (int) price;     // 19 (дробная часть отброшена!)
```

> [!caution] Потеря данных
> `(int) 19.99` даёт `19`. Десятичная часть пропадает без предупреждения.

## Операторы

### Арифметические

| Оператор | Значение | Пример |
|---|---|---|
| `+` | Сложение | `5 + 3` → `8` |
| `-` | Вычитание | `5 - 3` → `2` |
| `*` | Умножение | `5 * 3` → `15` |
| `/` | Деление | `10 / 3` → `3` (целое!) |
| `%` | Остаток | `10 % 3` → `1` |

> [!warning] Деление целых чисел
> `10 / 3` даёт `3`, а не `3.33`. Используй `10 / 3.0` для `3.333...`.

### Сравнения и логические

| Оператор | Значение |
|---|---|
| `==` `!=` | Равно / Не равно |
| `>` `<` `>=` `<=` | Больше, меньше и т.д. |
| `&&` `||` `!` | И, ИЛИ, НЕ |

> [!caution] `==` для String
> Используй `.equals()` для сравнения содержимого: `"hello".equals("hello")` → `true`.

## Основы String

```java
String name = "Alice";
System.out.println(name.length());                  // 5
System.out.println(name.charAt(0));                // 'A'
System.out.println(name.substring(1, 4));          // "lic"
System.out.println(name.equals("Alice"));          // true
System.out.println(name.toUpperCase());            // "ALICE"
System.out.println(name.contains("lic"));          // true
```

> [!info] Неизменяемость строк
> Строки нельзя изменить после создания. Каждое "изменение" создаёт новый объект. Для активной работы используй `StringBuilder`.

## var (Java 10+)

Компилятор определяет тип сам. Переменная всё ещё строго типизирована.

```java
var name = "Alice";          // String
var age = 25;                // int
var numbers = new int[5];    // int[]

// var name;                 // ❌ Нужно сразу инициализировать
```

> [!tip] Когда использовать
> Используй, когда тип очевиден: `var list = new ArrayList<String>();`. Избегай, когда непонятно: `var result = someMethod();`.

## Классы-обёртки и автоупаковка

Дженерики и коллекции не работают с примитивами (`List<int>` недопустим). Java предоставляет **класс-обёртку** для каждого примитивного типа — класс, хранящий примитивное значение как объект.

| Примитив | Обёртка | Пример |
|---|---|---|
| `int` | `Integer` | `Integer num = 42;` |
| `double` | `Double` | `Double pi = 3.14;` |
| `boolean` | `Boolean` | `Boolean flag = true;` |
| `char` | `Character` | `Character ch = 'A';` |
| `long` | `Long` | `Long big = 100L;` |
| `byte` | `Byte` | `Byte b = 1;` |
| `short` | `Short` | `Short s = 10;` |
| `float` | `Float` | `Float f = 2.5f;` |

### Автоупаковка и распаковка

Java конвертирует между примитивами и обёртками автоматически:

```java
// Автоупаковка (autoboxing): примитив → обёртка (автоматически)
Integer num = 42;             // компилятор делает: Integer.valueOf(42)
List<Integer> list = new ArrayList<>();
list.add(5);                  // автоупаковка 5 → Integer.valueOf(5)

// Распаковка (unboxing): обёртка → примитив (автоматически)
int x = num;                  // компилятор делает: num.intValue()
int first = list.get(0);     // авто-распаковка Integer → int
```

### Ловушка кеша Integer

Java кеширует значения `Integer` от **-128 до 127**. В этом диапазоне `Integer.valueOf()` возвращает один и тот же объект. За пределами диапазона — каждый раз создаёт новый.

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);    // true — один закешированный объект

Integer c = 128;
Integer d = 128;
System.out.println(c == d);    // false — разные объекты!
System.out.println(c.equals(d)); // true — одинаковое значение
```

> [!danger] Всегда используй .equals() для обёрток
> Никогда не сравнивай обёртки через `==`. Для малых чисел (кеш) работает, для больших — нет. Всегда используй `.equals()`.

### Опасность null

Обёртки могут быть `null`, примитивы — нет. Распаковка `null` выбрасывает `NullPointerException`:

```java
Integer value = null;
int x = value;           // NullPointerException! (пытается вызвать value.intValue())
```

Это очень частый баг при работе с коллекциями, результатами из БД или парсинге JSON — любой из этих источников может вернуть `null` вместо числа.

> [!tip] Когда что использовать
> Используй **примитивы** для локальных вычислений, счётчиков циклов и везде, где `null` не нужен. Используй **обёртки** когда `null` — допустимое значение (например, "данных нет"), в коллекциях (`List<Integer>`) и в обобщённом коде.

## Вопросы для собеседования

### Вопросы начального уровня

**1. Сколько примитивных типов в Java? Назовите их.**
В Java 8 примитивных типов: `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`. Они хранят значения напрямую в памяти, а не как объекты.

**2. В чём разница между примитивными и ссылочными типами?**
Примитивы хранят само значение в памяти. Ссылочные типы хранят *адрес в памяти*, указывающий на объект. Примитивы сравниваются по значению (`==`), а для ссылочных типов `==` сравнивает адреса, а не содержимое.

**3. Что такое приведение типов? Когда нужно явное приведение?**
Приведение типов конвертирует один тип данных в другой. Неявное приведение (расширение) происходит автоматически при переходе от меньшего типа к большему (`int → long`). Явное приведение (сужение) требуется при переходе от большего типа к меньшему (`double → int`) и может терять данные.

**4. В чём разница между `==` и `.equals()` для String?**
`==` сравнивает адреса в памяти — он проверяет, указывают ли две ссылки на один объект. `.equals()` сравнивает фактическое содержимое строк. Для сравнения строк всегда используй `.equals()`.

**5. Чему равно `10 / 3` в Java? Почему?**
Результат `3`, а не `3.333`, потому что оба операнда — целые числа. Java выполняет целочисленное деление, отбрасывая дробную часть. Чтобы получить `3.333`, хотя бы один операнд должен быть числом с плавающей точкой: `10 / 3.0` или `(double) 10 / 3`.

**6. В чём разница между `++x` и `x++`?**
`++x` (пре-инкремент) сначала увеличивает значение, а затем возвращает новое. `x++` (пост-инкремент) сначала возвращает текущее значение, а затем увеличивает. Пример: если `x = 5`, то `++x` вернёт 6, а `x++` вернёт 5 и x станет 6.

**7. В чём разница между `float` и `double`?**
`float` — 32 бита (точность ~7 десятичных знаков), требует суффикса `f`. `double` — 64 бита (точность ~15 десятичных знаков), используется по умолчанию для дробных литералов. Для большинства расчётов предпочтительнее `double`.

### Вопросы среднего уровня

**1. Что такое ключевое слово `var`? Когда его использовать?**
`var` (появилось в Java 10) позволяет компилятору вывести тип переменной из правой части присваивания. Используй его, когда тип очевиден (`var list = new ArrayList<String>()`). Избегай, когда тип неясен (`var result = getData()` — какой тип у `result`?).

**2. Почему String неизменяемый в Java?**
Строки неизменяемы ради безопасности (чувствительные данные вроде паролей), кеширования (String pool), потокобезопасности (не нужна синхронизация) и производительности (кеширование хэш-кода).

**3. Что такое составные операторы присваивания? Приведите примеры.**
Составные операторы совмещают операцию с присваиванием: `+=`, `-=`, `*=`, `/=`, `%=`. Пример: `x += 5` эквивалентно `x = x + 5`. Они также включают неявное приведение типов: `short s = 5; s += 2;` компилируется, а `s = s + 2;` — нет.

**4. В чём разница между локальной (local), instance и class (static) областью видимости?**
Локальная — объявлена внутри метода/блока, видна только там. Instance — поле, по одной копии на каждый объект. Class (static) — поле, общее для всех объектов класса.

### Вопросы повышенного уровня

**1. Чему равно `2.0 - 1.1` в Java? Почему?**
Это не `0.9`. Получится что-то вроде `0.8999999999999999` из-за погрешностей плавающей точки. Двоичное представление не может точно представить `0.1`. Для точных десятичных вычислений (деньги, расчёты) всегда используй `BigDecimal`.

**2. Можно ли использовать `var` для параметров методов или возвращаемых типов?**
Нет. `var` можно применять только к локальным переменным внутри методов. Его нельзя использовать для параметров методов, возвращаемых типов, полей или параметров в `catch`.

**3. Почему в Java нет примитивного типа для строк?**
Создатели Java решили сделать String классом ради гибкости: его можно интернировать (String Pool для оптимизации памяти), он может иметь методы вроде `length()`, `substring()` и поддерживать контракт неизменяемости. Примитивы не могут иметь методов или участвовать в String Pool.

**4. Что произойдёт при касте большого значения `long` в `int`?**
Если значение выходит за диапазон `int` (−2³¹ до 2³¹−1), биты обрезаются и результат заворачивается (overflow). Java не выбрасывает исключение. Пример: `(int) 3_000_000_000L` даст отрицательное число.

**5. Как кеш Integer (от -128 до 127) влияет на `==` для обёрток?**
В пределах кеша `Integer.valueOf()` возвращает один и тот же объект, поэтому `==` даёт `true`. Вне кеша создаются новые объекты, и `==` даёт `false`. Для обёрток всегда используй `.equals()`.

### Вопросы с кодом

**1. Что выведет этот код и почему?**

```java
Integer a = 127;
Integer b = 127;
Integer c = 128;
Integer d = 128;
System.out.println(a == b);
System.out.println(c == d);
System.out.println(c.equals(d));
```

Вывод: `true`, `false`, `true`. `a` и `b` попадают в кеш Integer (-128..127) → один объект → `==` работает. `c` и `d` вне кеша → разные объекты → `==` не работает, но `.equals()` сравнивает содержимое, поэтому `true`.

**2. Скомпилируется ли это? Что произойдёт при выполнении?**

```java
Integer value = null;
int x = value;
```

Компилируется. При выполнении выбрасывает `NullPointerException` — распаковка `null` вызывает `value.intValue()` на `null`. Частый баг при чтении из коллекций, результатов БД или парсинге JSON.

**3. Что выведет этот код?**

```java
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;
arr2[0] = 99;
System.out.println(arr1[0]);
```

Выведет `99`. Массивы — ссылочный тип: `arr2 = arr1` копирует адрес, обе переменные указывают на один массив. Изменение через одну затрагивает другую.
